# IOS-messenger designed using Firebase

While desiging the application, we used the following patters:
- Model-View-Controller;
- Solid: single responsibility principle, open/closed principle, liskov substitution principle, interface segregation principle, dependency inversion principle;
- GRASP: information expert, creator, low coupling, high cohesion, controller, polymorphism, pure fabrication, indirection, protected variations;
- Singleton;
- Observer; 
- so on. <br><br>

A major part of designing the application was integration with Google's Firebase. It is a platform which provides us with realtime database and storage. It is also responsible for authorization.
